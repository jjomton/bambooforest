### Slide 5. 시스템 아키텍처 및 익명 보안 (Security Architecture)
*   **슬라이드 제목:** 04. 보안의 핵심: 섀도 매핑 테이블을 통한 익명성 보장
*   **핵심 내용 (Bullet Points):**
    *   **일반적인 문제:** 게시글에 작성자 고유 ID가 노출될 경우, API 탈취 시 익명성 붕괴 위험.
    *   **해결책 (섀도 매핑 패턴):**
        1. `posts`(게시글) 테이블에는 작성자 정보가 없음.
        2. 글 작성 시 DB 내부 트리거가 작동하여 보안 격리된 `post_authors`에 `(글 ID, 유저 ID)` 비밀 매핑.
        3. RLS(Row Level Security) 정책을 통해 본인 소유 테이블만 수정/삭제 권한 부여.
    *   **효과:** 해킹 위협에도 익명성이 수학적으로 철저히 차단되며, 수정 및 삭제 권한은 완벽 통제.
*   **발표 스크립트 (Speaker Notes):**
    > *"익명 게시판에서 가장 중요한 것은 보안입니다. 저희 팀은 '섀도 매핑 테이블'과 Supabase RLS 정책을 유기적으로 연동하여, 네트워크 상에서 작성자 신원이 절대 유출될 수 없도록 설계했습니다."*

### Slide 6. 핵심 기능 및 자정 작용 (Moderation & Clean Community)
*   **슬라이드 제목:** 05. 텍스트 필터링 및 자동 블라인드 엔진
*   **핵심 내용 (Bullet Points):**
    *   **실시간 비속어 차단:** `profanityFilter.ts` 정규식 검사를 통해 게시글 및 댓글 작성 시 욕설 입력을 원천 차단.
    *   **3회 신고 누적 자동 블라인드:**
        *   수강생이 불쾌한 글을 신고하면 `reports` 테이블에 자동 집계.
        *   신고 3회 도달 시 PostgreSQL 트리거 `public.handle_auto_blind()` 발동.
        *   `is_blinded = true` 처리되어 피드에서 자동 숨김 및 관리자 검토 대기열로 이동.
*   **발표 스크립트 (Speaker Notes):**
    > *"건전한 대나무숲을 유지하기 위해 1차적으로는 정규식 기반의 비속어 차단을 적용했고, 2차적으로는 3회 신고 시 데이터베이스 트리거에 의해 즉각 블라인드 처리되는 자정 시스템을 구축했습니다."*

### Slide 7. 트러블 슈팅 사례 (Troubleshooting)
*   **슬라이드 제목:** 06. 난관 극복: 주요 장애 분석 및 조치 이력
*   **핵심 내용 (Bullet Points):**
    *   **PowerShell 권한 장애:** Windows 보안 정책으로 스크립트 실행 불가 ➡️ `-ExecutionPolicy Bypass` 옵션으로 해결.
    *   **Notion API 404 장애:** Notion 봇이 데이터베이스를 찾지 못함 ➡️ 데이터베이스 우측 상단 `연결 추가`로 봇 권한 활성화하여 동기화 성공.
    *   **Vite 빌드 타입 불일치:** Mock 데이터의 `is_blinded` 프로퍼티 누락 ➡️ 데이터 구조 정의에 기본값 추가하여 타입 안정성 확보.
*   **발표 스크립트 (Speaker Notes):**
    > *"로컬 스크립트 권한 문제부터 API 권한 연동 및 TypeScript 빌드 에러까지 다양한 난관이 있었으나, 정확한 원인 분석을 통해 해결했으며, 이를 모두 노션 트러블 슈팅 DB에 아카이빙하여 개발팀의 소중한 자산으로 만들었습니다."*
